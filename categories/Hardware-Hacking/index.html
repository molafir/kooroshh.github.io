AirMaster AG3100D Security Analysis

AG3100D is the newest variant of 3100 families with the support of 4G/LTE.
the whole firmware has changed (probably the board has changed too, I haven’t opened the device box :/)
there are some improvements on the new firmware, but most of the previous enhancements are lost due to making the latest firmware.

Password Hashing (good, bad, and the ugly)

user passwords are hashed in this firmware using HMAC-SHA1, but the problem is, it’s getting hashed on the client-side, and the hash gets checked in the embedded web server against the hashed password stored in the NVRAM.
you can read the javascript hashing code

The good part is the attacker can’t read the user’s passwords, but thanks to hashing in the client-side, he can replace his passwords if he can get access from NVRAM!

Insecure User Role

like the previous versions of the firmware, most of the pages in the admin interface are invisible for the Guest user (admin), at the same time CGI handler won’t check the user access role in the incoming requests, so if you have knowledge about the firmware and it’s HTTP requests, you can modify the whole settings with the lowest user role.

for example, the following URL can be accessed through the Guest Role to change the Sim Lock functionality
http://192.168.1.1/lte/ScanOperator.asp

Insecure Root Password

the /etc/passwd contains a BSD hashed password: $1$0kUTQMej$d1Gag3dRMwvuVjlYSsHIf.
after 12 hours of brute-forcing, the password was root123 !!!
so the telnet/ssh credentials are the root/root123, which is so insecure.
(honestly, I blame myself why I didn’t check the dictionary and waste 12 hours of my GPU power =) )

AirMaster TD-LTE CPE Security Analysis

Jan 21, 2021 in Hardware Hacking, Security

AirMaster is a product family of KZ Broadband Technology “KZTech”.
There are numerous models such as 3000,3100,3100e,3100v that use the same firmware base.

in this post, I will discuss the 3000 model with firmware version 2.0.1B1047.

There are tons of backdoor included in the firmware with different methods.
however, I think some of them are logical bugs.

Super Cookie

if we put the following cookie Cookie: kz_userid=Administrator:1 in the cookie jar, we can get access to the CPE without any password! it’s fun and scary at the same time but this device is a Chinese one so it’s normal :)

Deep Inspection of GoAhead


what is syscmd?
“syscmd.asp” is a page that you can execute shell commands! but it’s password protected and cookie protected (requires authentication)
as far as we had the Super Cookie we can bypass the authentication but what about the password ?! it’s quite simple, in this version the password is super1234, and boom! we can execute all commands exactly inside the web page!

what if the device was sim locked?
let’s dig into the binary!

2 lines refer to the SimCard Restrictions, an asp page, and a form handler that handles the content of that asp page.
if we open 192.168.1.1/lte/simcardinfo.asp using administrator credentials or the supercookie we can simply disable the restrictions!
if we open the page with admin (guest) credentials we should use the js command to show the hidden parts.
